# ストーリーテリング リリース設計

この Skill は、Morimorimori の文言と体験を、VR 遠景制作の実運用品質で整えるときに使います。

最初に `../../AGENTS.md` を読みます。
そのあと `../../docs/storytelling-release-playbook.md` を基準として参照します。
必要に応じて次も併用します。

- `../pm-release-review/SKILL.md`
- `../ui-foundation-design/SKILL.md`
- `../frontend-component-implementation/SKILL.md`

## 目的

- サービス名から各状態の文言まで、一貫した語り口を作る
- 視点再現（観測点/対象点）を UI 状態遷移に統合する
- リリース可否を判定できる指標を定義する

## 実行手順

1. サービス価値を 1 文で定義する
2. ターゲットを「VR 背景地形制作者」として具体化する
3. 観測点/対象点/視点設定（高度・FOV・方位）を要件化する
4. 必須 7 状態を文言・CTA・表示まで定義する
5. 文言の禁止事項を確認し、混在や曖昧語を除去する
6. 大気感/skybox の比較導線を設計する
7. 定性・定量のリリース指標を設定する
8. デスクトップ/モバイルで導線を検証し、`ship/no-ship` を判定する

## 7 状態の必須出力

次の状態を必ず扱います。

- `splash`
- `idle`
- `searching`
- `ready-to-generate`
- `generating`
- `generated`
- `error`

各状態ごとに、必ず次を出力します。

- 表示内容
- 主 CTA
- 状態文言
- デスクトップでの見え方
- モバイルでの見え方

## 追加で必須化する VR 出力

- 観測点と対象点の定義
- 視点再現パラメータ（高度・FOV・方位）
- 大気感/skybox の適用方針
- 遠景向け出力時の座標系とスケール説明

## 文言ルール

- ユーザー向け文言は日本語を基本にする
- 主 CTA は短い命令形にする
- エラー文言は原因・影響・次行動を含める
- 「何が起きているか」「次に何をするか」を同時に示す

## 禁止事項

- 日英 UI の混在
- 説明不能な空白状態
- 見えない状態表示
- 意図不明の CTA
- 途中経過コメントや未整理の痕跡を残すこと

## 出力形式

返す内容は次の順序にします。

1. サービス定義
2. ターゲット定義
3. 機能要件（広域地形、視点再現、大気/skybox、連携）
4. 状態別ストーリー設計（7 状態）
5. 文言システム（テンプレートと禁止事項）
6. リリース指標（定性・定量）
7. リリース判定（`ship` / `no-ship`）
8. 最低限の修正項目
