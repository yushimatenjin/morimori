# デザインシステム統一ガバナンス

## 目的

- 画面ごとの場当たりUIを止め、統一された体験を維持する
- デザイナー視点の整合性を、実装時に崩さない

## 1. 統一ルール

1. 同じ役割のUIは同じコンポーネントを使う
2. 新しい見た目を作る前に既存バリアントで吸収できるか確認する
3. 1回しか使わない専用クラスは禁止
4. 状態表現は `data-state` と文言の両方で示す

## 2. コンポーネント優先順位

- Tier-1（主導線必須）: `Button`, `Input`, `Card`, `Badge`
- Tier-2（条件調整）: `Select`, `Slider`, `Switch`
- Tier-3（任意導線）: 仕上げ系コンポーネント

Tier-1 は破壊的変更禁止。変更時は全画面監査が必須。

## 3. デザイントークン運用

- 色: 既存トークンのみ使用（新色はデザインレビュー必須）
- タイポ: 見出し/本文/補助の3段階を超えない
- 余白: 4/8/12/16/24 のスケールで統一
- 角丸: 主要は `rounded-md` か `rounded-xl` に限定

状態色の契約（2026-03-15 更新）:

- `data-state` でアクセント色を切替える
- `idle/ready-to-generate`: `Terrain Cyan`
- `searching/generating`: `Sun Amber`
- `generated`: `Forest Emerald`
- `error`: `Alert Vermilion`
- 常時同色アクセントは禁止（主CTA優先度が判別できないため）

CTA配置契約:

- デスクトップ: サイドバー下部に主CTAドックを固定
- モバイル: 下部固定CTAを主CTAに同期
- 両者の文言・活性/非活性条件は一致させる

## 4. 変更審査（デザイナー）

変更前に次を確認:

- 主CTAが最も強いか
- 任意機能が主導線より目立っていないか
- モバイルで主CTAが初期表示内にあるか
- 文言だけで現在地と次行動がわかるか

1つでもNoなら差し戻し。

## 5. 廃止判断

以下を満たす要素は削除候補:

- 主導線3ステップへ直接寄与しない
- 初期表示で見えない
- 見えても次行動を変えない
