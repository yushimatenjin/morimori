# 結合テスト表（UI導線 / Playwright）

## 目的

- 主導線「ギャラリー選択 → 自動生成 → 保存」が壊れていないことを確認する
- 全国主要地点のサムネイル選択で即生成できることを確認する
- 手動検索は任意導線として機能し、主導線を阻害しないことを確認する
- 状態遷移ごとのフィードバック文言が成立していることを確認する
- 機能面（生成・保存・復帰）の実測値がレビュー可能であることを確認する

## テストケース

| ID | シナリオ | 操作 | 期待結果 | フィードバック文言 |
|---|---|---|---|---|
| IT-01 | 初回起動から作業開始 | Splashで`制作を始める` | 状態が`idle`へ遷移 | ギャラリー選択または手動検索を案内 |
| IT-02 | ギャラリー→自動生成 | ギャラリーで地点選択（例: 富士山） | 状態が`generated`、主CTAが`3Dモデルを保存` | 自動表示完了と次行動を案内 |
| IT-03 | 検索未入力エラー | 空入力で検索 | 状態が`error` | `原因/影響/次` を含む文言 |
| IT-04 | 手動検索導線 | `手動で地点を検索` を開く→地点検索 | 状態が`generated`、主CTAが`3Dモデルを保存` | 検索結果の自動表示を案内 |
| IT-05 | 範囲調整→再生成 | `中域 18km` などを調整して再生成 | 再生成後も `generated` に戻る | 条件変更が反映される |
| IT-06 | 地点未選択ガード | 開始直後に主CTA押下 | 生成されず `idle` 維持 | 先に地点選択する案内が出る |
| IT-07 | モバイル導線 | 390x844で開始→ギャラリー選択 | ギャラリーカードと固定主CTAが可視 | 生成までの次行動が読める |
| IT-08 | 入力負荷抑制 | 開始直後に`手動で地点を検索`表示確認 | 検索入力は折りたたみ、開いた時のみ表示 | まず地点選択の導線が見える |
| IT-09 | 主CTA固定表示 | 1440x900 / 390x844 で開始 | `PrimaryActionSticky` が常に表示 | 主導線の主操作を見失わない |
| IT-10 | 生成後の任意導線 | 生成完了後 | `OptionalFinishingSection` が表示される | 任意導線は生成後のみ開く |
| IT-11 | 連打ガード | 生成中にギャラリーを連打 | 重複生成が走らない | 進行中の文言が継続される |
| IT-12 | 機能レビュー表示 | `?review=1` で開始 | `FunctionalReviewSection` が表示される | 生成/保存/失敗の数値を確認できる |
| IT-13 | 必要度スコア変動 | 空検索でエラー→地点選択で再生成 | 必要度が0〜100%内で更新される | エラー増加時に上昇し、その後の操作で再計算される |
| IT-14 | 差分スコア保存 | `現在値を基準保存`→操作実行 | 機能ごとの差分（±%）が表示される | 改善/悪化の方向が判別できる |

## フィードバック判定ルール

- 1件でも Fail があれば `no-ship`。主導線に関係するケース（IT-01/02/05/06/07）は特に即修正。
- 文言が期待とズレる場合は `Warn` として記録し、次回修正候補に積む。
- Playwright 実操作の結果を優先し、`typecheck/build` は補助確認として扱う。
- リグレッション修正時は、必ず対象ケースと隣接ケース（最低2件）を再実行する。
